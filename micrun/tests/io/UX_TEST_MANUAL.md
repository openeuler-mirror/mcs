# MicRun UX 测试指南

## 手动测试步骤

请按照以下步骤在测试环境 `root@192.168.7.2` 上执行测试：

### 测试 1：双重回显修复验证 ⭐ 重点测试

此测试验证 "输入1次help等命令会有2次显示" 问题的修复效果。

```bash
# 1. 连接到测试环境
ssh -tt root@192.168.7.2

# 2. 清理日志（可选，方便后续调试）
rm -f /var/log/mica/mica-runtime.log

# 3. 启动 RTOS 容器
nerdctl run -it --runtime io.containerd.mica.v2 --network=none \
  -l org.openeuler.micrun.container.auto_close=false \
  localhost:5000/mica-uniproton-app:xen-0.1

# 4. 等待提示符出现（如 "openEuler UniProton #" 或 MainTCB>）

# 5. 输入 help 命令并观察回显行为
help

# 6. 关键观察点：
#    ✅ 修复前：输入 "help" 会显示 "helphelp"（双重回显）
#    ✅ 修复后：输入 "help" 只显示 "help" 一次
#    注意：这里说的是你输入时看到的回显，不包括 RTOS 的响应
```

**判定标准：**
- 当你逐个输入 `h` `e` `l` `p` 字符时
- 每个字符应该只显示一次（由 PTY 回显）
- 如果看到每个字符显示两次，说明修复未生效
- RTOS 的 help 响应内容是正常的，不属于双重回显问题

### 测试 2：多次命令一致性

```bash
# 1. 输入 help，记录输出
help

# 2. 按回车（空命令）

# 3. 再次输入 help，比较输出是否与第一次相同
help

# 4. 记录：
#    - 两次 help 输出是否完全一致？
#    - 空命令是否只显示提示符？
```

### 测试 3：快速连续输入

```bash
# 快速连续输入以下命令（不要等待响应）：
help
uname
vendor

# 观察：
#    - 每个命令的响应是否独立完整？
#    - 是否有字符乱序或丢失？
#    - 是否有重复输出？
```
